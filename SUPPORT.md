# Support

## 1. 获取帮助 | Get Help

- 使用问题 / How-to questions: GitHub Discussions（建议）
- 缺陷报告 / Bug reports: GitHub Issues（使用 bug 模板）
- 功能建议 / Feature proposals: GitHub Issues（使用 feature 模板）
- 安全漏洞 / Security vulnerabilities: `SECURITY.md` 指引流程

## 2. 提问前检查 | Before Opening an Issue

请先确认：
- 已阅读 `README.md` 与 `CONTRIBUTING.md`
- 已检查 `docs/prd.md` 与对应模块文档
- 已搜索现有 Issues/Discussions，避免重复

## 3. 高质量问题单 | High-Quality Support Requests

建议附带：
- 运行环境（OS/Go/Node/pnpm 版本）
- 执行命令与完整错误日志
- 最小复现步骤（minimum reproducible steps）
- 预期行为与实际行为

## 4. 响应预期 | Response Expectations

项目为开源协作，维护者会按优先级处理。  
This is an open-source project; maintainers respond based on priority and bandwidth.
