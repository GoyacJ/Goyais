# Goyais 产品需求文档（PRD）

## 1. 产品概述

**Goyais** 是一款面向开发者的 AI 协作开发平台，通过桌面客户端与远程工作区的双模式架构，帮助开发者在个人项目或团队协作中高效利用 AI 能力完成代码生成、调试与变更管理。

- **产品定位**：AI 优先的开发者协作工具，聚焦「对话即开发」的核心理念
- **核心价值主张**：以会话驱动开发流程，整合 AI Agent、MCP 工具链与版本控制，提供从需求到代码的端到端体验
- **目标用户**：开发者个人、研发团队、技术管理者
- **当前版本**：v0.4.0，MIT 开源协议

---

## 2. 产品架构概览

Goyais 采用三端分离架构：

| 端点 | 形态 | 职责 |
|---|---|---|
| **Desktop** | 桌面客户端（Electron + Vue 3） | 本地工作区入口，提供完整 UI 与本地执行能力 |
| **Mobile** | 移动端 App（iOS/Android） | 远程工作区接入，轻量化交互界面 |
| **Hub** | 后端服务（Go） | 身份认证、权限控制、API 契约、Agent 调度 |

### 两种工作区模式

- **本地工作区（Local Workspace）**：桌面端独占模式，数据与执行均在本地完成，适合个人开发者离线使用
- **远程工作区（Remote Workspace）**：通过 Hub 托管，支持团队成员共享访问，需网络连接

### 运行模型说明

用户通过 Desktop 客户端创建会话，向 AI Agent 发送自然语言需求；Agent 理解意图后调用 MCP 工具或直接操作代码，完成后生成变更集（diff）。所有操作均通过 Hub 统一调度，确保权限一致性与审计可追溯。

## 3. 核心功能模块

### 3.1 会话与对话（Session）

| 功能 | 描述 |
|---|---|
| **项目/对话管理** | 创建、选择、重命名、删除会话；支持 Markdown 导出 |
| **消息撰写器** | 支持 `@资源` 引用文件/代码片段；支持 `/命令` 快速调用技能；支持执行模式切换（Plan ↔ Execute） |
| **实时执行追踪（RunTrace）** | 可视化展示 Agent 推理过程、工具调用链与中间结果 |
| **工具调用确认** | 对危险操作（如文件删除、git push）进行 Approve/Deny 审批 |
| **对话回滚（Rollback）** | 回退到会话历史中的任意状态 |
| **消息排队（Queue）** | 连续发送多条消息时自动排队，FIFO 顺序处理 |

### 3.2 变更集与版本管理

| 功能 | 描述 |
|---|---|
| **文件 Diff 视图** | 逐文件展示 AI 生成的代码变更，支持行级对比 |
| **Commit / Discard** | 一键提交变更到 Git，或丢弃未保存的修改 |
| **Export Patch** | 导出标准 patch 文件供审查或分享 |
| **AI 生成 Commit Message** | Agent 自动分析变更内容，生成符合 Conventional Commits 规范的提交信息 |

### 3.3 工作区资源配置

| 功能 | 描述 |
|---|---|
| **AI 模型管理** | 支持多厂商模型：OpenAI、DeepSeek、Google (Gemini)、Qwen (阿里)、Doubao (字节)、Zhipu (智谱)、MiniMax；可配置 API Key、温度、最大 token 等参数 |
| **MCP 服务器** | 支持 stdio 与 HTTP 两种连接协议；支持批量导入/导出服务器配置 |
| **规则（Rules）** | Markdown 格式的上下文约束文档，可注入到 Agent 提示词 |
| **技能（Skills）** | 可复用的指令模板，支持参数化调用 |
| **Agent 执行参数** | 配置工具调用策略、超时时间、最大迭代次数等 |

### 3.4 本地设置

| 功能 | 描述 |
|---|---|
| **主题** | 三种预设：浅色、深色、系统跟随；支持自定义字体、字号 |
| **语言/国际化** | 支持中文、英文双语 UI |
| **通用设置** | 启动项、自定义工作目录、系统通知、遥测数据、更新策略、诊断日志 |

### 3.5 远程工作区管理（管理员）

| 功能 | 描述 |
|---|---|
| **账户信息** | 显示当前登录用户、连接状态、Hub 地址 |
| **成员与角色管理** | 邀请成员、分配角色（viewer/developer/approver/admin） |
| **权限审计** | 菜单可见性控制、ABAC 策略配置、操作审计日志 |

## 4. 用户流程

### 首次使用流程

```
1. 启动 Desktop 客户端
2. 导入本地项目目录（或创建新项目）
3. 创建新会话 → 输入需求
4. Agent 执行 → RunTrace 展示过程
5. 审阅 Diff → Approve/Deny 工具调用
6. 确认变更 → Commit 到 Git
```

### 工作区切换流程

- **本地 → 远程**：从工作区列表选择远程工作区，输入 Hub 地址与凭证，验证通过后切换
- **远程 → 本地**：在远程工作区页面点击「切换到本地」，自动回切

### 工具调用审批流

1. Agent 发起工具调用请求
2. 系统根据权限模式判断：
   - **默认模式**：低风险操作自动执行，高风险操作弹窗确认
   - **Plan 模式**：仅生成计划，不执行
   - **AcceptEdits 模式**：自动接受文件编辑，拒绝其他操作
   - **DontAsk 模式**：静默执行所有操作
   - **BypassPermissions**：管理员跳过权限检查

### 远程团队协作流

1. 管理员创建远程工作区，邀请团队成员
2. 成员通过 Desktop/Mobile 加入
3. 根据角色分配权限（viewer/developer/approver）
4. 所有操作记录到审计日志

## 5. 权限模型

### 四种角色

| 角色 | 权限描述 |
|---|---|
| **viewer** | 仅可查看会话、对话和变更，不可执行 AI 操作 |
| **developer** | 可发起 AI 执行、操作资源（模型/MCP/规则），不可管理成员 |
| **approver** | 额外拥有工具调用审批权限，可审核他人变更 |
| **admin** | 完整权限，包括成员管理、角色分配、权限审计、系统配置 |

### 三种能力标志

| 能力 | 说明 |
|---|---|
| **admin_console** | 访问管理员控制台（成员、角色、审计页面） |
| **resource_write** | 修改工作区资源配置（模型、MCP、规则、技能） |
| **execution_control** | 控制 AI Agent 执行（发起、审批、中断） |

### 权限继承规则

- 本地工作区默认拥有全部三种能力标志（角色固定为 admin）
- 远程工作区权限由 Hub 根据用户角色动态下发
- 权限模式通过菜单可见性（menu_visibility）与动作可见性（action_visibility）细粒度控制

## 7. 非功能需求

### 性能要求

- **前端**：lint / test / e2e:smoke 全部通过
- **后端**：go test / go vet 全部通过

### 安全要求

- **双层权限**：ABAC（属性访问控制）+ RBAC（角色访问控制）
- **Hook 策略**：可配置 Pre/Post Hook 拦截执行流
- **本地 Sandbox**：本地工作区代码执行在隔离环境中

### 国际化

- UI 语言：中文（zh-CN）、英文（en-US）
- 所有文本字段支持国际化键值替换

### 可扩展性

- OpenAPI 契约驱动开发
- TypeScript 类型自动生成（openapi.yaml → client types）
- MCP 协议支持第三方工具快速接入

## 8. 已知限制与未来规划

### 当前版本限制（v0.4.0）

- Mobile 端仅支持远程工作区，不支持本地模式
- Web 端仅预留接入能力，暂未开放
- 部分 MCP 服务器配置仅支持 stdio 协议
- 审计日志仅在远程工作区可用
