# Goyais v0.1 规格拆解（Epic -> Stories -> Tasks）

## 1. 范围与冻结前提

本规格基于 `docs/prd.md` 与仓库冻结决策：
- API 前缀固定 `/api/v1`。
- Command-first：AI 写操作必须走 `/commands`。
- Domain 写接口内部必须转 Command，并返回 `resource + commandRef`。
- 最小化运行：`SQLite + MediaMTX + local file store + memory cache`。
- 生产发布：单二进制（Go embed 前端 dist），SPA fallback。
- `index.html` 缓存头固定 `Cache-Control: no-store`。
- `/favicon.ico`、`/robots.txt` 缺省 404，不走 fallback。

## Epic A：平台基座与统一契约

### Story A1：配置系统与 Provider 选择器

**目标**
- 提供统一配置加载（ENV/YAML），支持最小化与完整模式切换。

**Provider 支持**
- DB：`sqlite`、`postgres`
- Object Store：`local`、`minio`、`s3`
- Cache：`memory`、`redis`
- Vector：`sqlite`、`redis_stack`

**配置项（ENV/YAML）与默认值**
- `GOYAIS_DB_DRIVER` ↔ `db.driver`，默认 `sqlite`
- `GOYAIS_CACHE_PROVIDER` ↔ `cache.provider`，默认 `memory`
- `GOYAIS_VECTOR_PROVIDER` ↔ `vector.provider`，默认 `sqlite`
- `GOYAIS_OBJECT_STORE_PROVIDER` ↔ `object_store.provider`，默认 `local`
- `GOYAIS_STREAM_PROVIDER` ↔ `stream.provider`，默认 `mediamtx`
- `GOYAIS_CONFIG_FILE` ↔ `config.file`，默认 `./config.yaml`
- 优先级：`ENV > YAML > 默认值`

**API 端点（写入 openapi.yaml）**
- `GET /api/v1/system/config/effective`（管理员）
- `GET /api/v1/system/healthz`

**数据表/字段（写入 data-model.md）**
- `system_settings(key,value,scope,updatedBy,updatedAt)`（可选，v0.1 可只文档化）

**Tasks**
1. 定义配置 schema 与 provider 枚举。
2. 定义 profile：`minimal`、`full`。
3. 定义配置校验与错误键名。

**测试点与验收步骤**
- 切换 ENV 能覆盖 YAML。
- 配置非法值返回 `error.code=INVALID_CONFIG`。

---

### Story A2：Command Gate 与统一执行链

**目标**
- 建立 Validate/Authorize/Execute/Audit 统一执行管道。

**Provider 支持**
- DB：`sqlite`、`postgres`（命令与审计持久化）
- Cache：`memory`、`redis`（幂等键、短期状态缓存）
- Object Store/Vector：不直接依赖，但需在 command payload 可引用。

**配置项与默认值**
- `GOYAIS_COMMAND_IDEMPOTENCY_TTL` ↔ `command.idempotency_ttl`，默认 `300s`
- `GOYAIS_COMMAND_MAX_CONCURRENCY` ↔ `command.max_concurrency`，默认 `32`

**API 端点**
- `POST /api/v1/commands`
- `GET /api/v1/commands/{commandId}`
- `GET /api/v1/commands`

**数据表/字段**
- `commands(id,tenant_id,workspace_id,owner_id,type,status,payload,result,error_code,message_key,accepted_at,finished_at,created_at,updated_at)`
- `command_events(id,command_id,event_type,payload,created_at)`（可选）

**Tasks**
1. 定义 Command schema（asset/workflow/plugin/stream）。
2. 定义命令状态机：`accepted/running/succeeded/failed/canceled`。
3. 定义 Domain 写接口到 Command 的映射约束。

**测试点与验收步骤**
- 同一业务动作经 AI/UI 触发时 command type 与 payload 结构一致。
- Domain 写接口响应包含 `commandRef`。

---

### Story A3：Visibility/ACL 与授权闸门

**目标**
- 全对象统一 visibility/ACL 判定，支持 agent-as-user。

**Provider 支持**
- DB：`sqlite`、`postgres`（ACL 存储）
- Cache：`memory`、`redis`（策略缓存）
- 其他 provider：不限定。

**配置项与默认值**
- `GOYAIS_AUTHZ_POLICY_CACHE_TTL` ↔ `authz.policy_cache_ttl`，默认 `30s`
- `GOYAIS_ALLOW_PRIVATE_TO_PUBLIC` ↔ `authz.allow_private_to_public`，默认 `false`

**API 端点**
- `POST /api/v1/shares`
- `DELETE /api/v1/shares/{shareId}`
- `GET /api/v1/shares`

**数据表/字段**
- `acl_entries(id,resource_type,resource_id,subject_type,subject_id,permissions,expires_at,created_by,created_at)`
- 对象通用字段：`tenant_id/workspace_id/owner_id/visibility`

**Tasks**
1. 定义授权顺序：Tenant -> Visibility -> ACL -> RBAC -> Egress。
2. 定义 PUBLIC 发布权限约束。
3. 定义输出可见性继承策略字段。

**测试点与验收步骤**
- 未授权访问返回 `FORBIDDEN` + `messageKey`。
- PRIVATE 输入默认不允许产出 PUBLIC。

## Epic B：资产与工作流闭环

### Story B1：Asset 与血缘

**目标**
- 支持多模态资产上传、查询、共享与 lineage 记录。

**Provider 支持**
- DB：`sqlite`、`postgres`
- Object Store：`local`、`minio`、`s3`
- Cache：`memory`、`redis`（上传会话、短期元数据）
- Vector：可选（用于索引扩展）

**配置项与默认值**
- `GOYAIS_OBJECT_STORE_PROVIDER` ↔ `object_store.provider`，默认 `local`
- `GOYAIS_OBJECT_STORE_BUCKET` ↔ `object_store.bucket`，默认 `goyais-local`
- `GOYAIS_ASSET_MAX_SIZE_MB` ↔ `asset.max_size_mb`，默认 `1024`

**API 端点**
- `POST /api/v1/assets`
- `GET /api/v1/assets/{assetId}`
- `GET /api/v1/assets`
- `PATCH /api/v1/assets/{assetId}`
- `DELETE /api/v1/assets/{assetId}`
- `GET /api/v1/assets/{assetId}/lineage`

**数据表/字段**
- `assets(id,tenant_id,workspace_id,owner_id,visibility,name,type,mime,size,uri,hash,metadata,status,created_at,updated_at)`
- `asset_lineage(id,source_asset_id,target_asset_id,run_id,step_id,relation,created_at)`

**Tasks**
1. 定义资产 schema 与 metadata 占位对象。
2. 定义 lineage 边类型。
3. 定义 visibility + ACL 与查询过滤关系。

**测试点与验收步骤**
- 三种存储 provider 上传/读取一致。
- lineage 可从输出反查输入。

### Story B2：WorkflowTemplate（Draft/Publish/Patch）

**目标**
- 支持 DAG 模板草稿、发布快照、受控 patch。

**Provider 支持**
- DB：`sqlite`、`postgres`
- Cache：`memory`、`redis`（编辑会话、校验缓存）

**配置项与默认值**
- `GOYAIS_WORKFLOW_MAX_NODES` ↔ `workflow.max_nodes`，默认 `200`
- `GOYAIS_WORKFLOW_VALIDATE_STRICT` ↔ `workflow.validate_strict`，默认 `true`

**API 端点**
- `POST /api/v1/workflow-templates`
- `GET /api/v1/workflow-templates/{templateId}`
- `GET /api/v1/workflow-templates`
- `POST /api/v1/workflow-templates/{templateId}:patch`
- `POST /api/v1/workflow-templates/{templateId}:publish`

**数据表/字段**
- `workflow_templates(id,tenant_id,workspace_id,owner_id,visibility,name,description,status,current_version,graph,schema_inputs,schema_outputs,ui_state,created_at,updated_at)`
- `workflow_template_versions(id,template_id,version,graph,schema_inputs,schema_outputs,checksum,created_by,created_at)`

**Tasks**
1. 定义 patch schema（add/remove/update node/edge）。
2. 定义发布前校验规则（无环、类型匹配、必填输入）。
3. 定义不可变发布版本约束。

**测试点与验收步骤**
- 非法连线/成环无法发布。
- patch 变更可审计。

### Story B3：WorkflowRun/StepRun 执行与回放

**目标**
- 支持模板运行、step 追踪、取消/重试、回放查询。

**Provider 支持**
- DB：`sqlite`、`postgres`
- Cache：`memory`、`redis`（运行态缓存、队列占位）
- Object Store：产物持久化（`local/minio/s3`）

**配置项与默认值**
- `GOYAIS_RUN_MAX_RETRIES` ↔ `run.max_retries`，默认 `3`
- `GOYAIS_RUN_STEP_TIMEOUT_SEC` ↔ `run.step_timeout_sec`，默认 `600`

**API 端点**
- `POST /api/v1/workflow-runs`
- `GET /api/v1/workflow-runs/{runId}`
- `GET /api/v1/workflow-runs`
- `POST /api/v1/workflow-runs/{runId}:cancel`
- `GET /api/v1/workflow-runs/{runId}/steps`

**数据表/字段**
- `workflow_runs(id,tenant_id,workspace_id,owner_id,visibility,template_id,template_version,status,inputs,outputs,error_code,message_key,started_at,finished_at,created_at,updated_at)`
- `step_runs(id,run_id,step_key,step_type,status,attempt,input,output,log_ref,error_code,message_key,started_at,finished_at,created_at,updated_at)`

**Tasks**
1. 定义 run/step 状态机与重试规则。
2. 定义 `run from here` 与 replay 查询契约。
3. 定义运行产物与 lineage 绑定。

**测试点与验收步骤**
- 失败重试后状态正确。
- cancel 可将 running 状态收敛到 canceled。

## Epic E：前端基线与发布形态

### Story E1：前端基线（Vue/Vite/TS/Tailwind + i18n + theme）

**目标**
- 建立前端框架基线，支持国际化与深浅色模式。

**Provider 支持**
- 不直接依赖基础设施 provider；但必须兼容后端错误 `messageKey`。

**配置项与默认值**
- `GOYAIS_WEB_DEFAULT_LOCALE` ↔ `web.default_locale`，默认 `zh-CN`
- `GOYAIS_WEB_SUPPORTED_LOCALES` ↔ `web.supported_locales`，默认 `zh-CN,en-US`
- `GOYAIS_WEB_THEME_DEFAULT` ↔ `web.theme_default`，默认 `system`

**API 端点**
- 依赖通用查询端点，无新增强制写端点。

**数据表/字段**
- 可选：`user_preferences(user_id,locale,theme,updated_at)`。

**Tasks**
1. 定义 i18n key 命名规范。
2. 定义错误 messageKey 映射兜底策略。
3. 定义主题切换状态存储策略。

**测试点与验收步骤**
- `zh-CN/en-US` 切换即时生效。
- 主题切换后页面样式一致，无关键组件不可读。

### Story E2：单二进制打包与嵌入式静态服务

**目标**
- 生产环境仅使用单二进制提供 API 与前端静态内容。

**Provider 支持**
- 与最小化运行一致：`sqlite + memory + local + mediamtx` 需可独立运行。

**配置项与默认值**
- `GOYAIS_WEB_EMBED_ENABLED` ↔ `web.embed_enabled`，默认 `true`
- `GOYAIS_WEB_INDEX_CACHE_CONTROL` ↔ `web.index_cache_control`，默认 `no-store`

**API/路由端点**
- API：`/api/v1/*`
- 静态：`/assets/*`（embed）
- SPA fallback：`/`、`/canvas` 等前端路由
- 特殊：`/favicon.ico`、`/robots.txt` 缺省 404

**数据表/字段**
- 无新增强制表；属于访问层行为规范。

**Tasks**
1. 定义路由优先级：API > 静态文件 > favicon/robots 策略 > fallback。
2. 定义 `index.html` 缓存头固定 `Cache-Control: no-store`。
3. 定义静态 `Content-Type` 最低要求。
4. 定义“无外部 dist 依赖”验收脚本步骤。

**测试点与验收步骤**
- `make build` 后删除/改名 `web/dist`（甚至 `web/`），二进制仍能服务 `/`、`/canvas`、`/api/v1/healthz`。
- `/` 与 `/canvas` 返回 `Cache-Control: no-store`。
- `assets/*.js` 返回正确 JavaScript `Content-Type`。
- 无占位文件时 `/favicon.ico`、`/robots.txt` 返回 404。

## 2. 跨 Story 公共契约

### 2.1 统一错误结构
```json
{
  "error": {
    "code": "FORBIDDEN",
    "messageKey": "error.authz.forbidden",
    "details": {}
  }
}
```

### 2.2 统一分页结构（Hybrid）
```json
{
  "items": [],
  "pageInfo": {
    "page": 1,
    "pageSize": 20,
    "total": 100
  },
  "cursorInfo": {
    "nextCursor": "opaque-token"
  }
}
```

### 2.3 通用对象字段（关键字段齐全、可扩展）
- 必需：`id tenantId workspaceId ownerId visibility acl createdAt updatedAt status`
- 可扩展：`metadata config params details` 使用 `object` 或 `additionalProperties`
