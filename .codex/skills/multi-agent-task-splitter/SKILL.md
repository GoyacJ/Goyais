# Multi-Agent 任务拆分技能

当任务可拆为互不依赖的并行子任务时使用本技能。

## 触发条件

- 至少两个子任务相互独立。
- 子任务执行不会产生共享可变状态冲突。

## 输入

- 总体目标
- 候选子任务列表
- 文件触达映射
- v0.4.0 权威约束

## 工作流

1. 拆解为独立单元并分配明确责任。
2. 为每个代理定义目标、输入、预期产物。
3. 设置边界避免重叠和冲突。
4. 执行最终整合与不变量校验。

## 输出契约

- `decomposition_plan`
- `agent_objectives`
- `inputs_per_agent`
- `merge_checklist`
- `integration_validation`

`merge_checklist` 必须包含：

- 重叠文件冲突检查
- 架构不变量检查
- 安全/风险确认检查
- 文档一致性影响检查

## 护栏

- 若任务并不独立，不得强行分派。
- 状态耦合高时优先单代理执行。
- 完成前必须有最终整合步骤。
- 对用户过程说明默认使用中文。
