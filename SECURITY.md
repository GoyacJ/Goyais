# Security Policy

## Supported Versions

The following versions currently receive security updates:

| Version | Supported |
| --- | --- |
| 0.2.x | Yes |
| < 0.2.0 | No |

## Reporting a Vulnerability

Please report security vulnerabilities using **GitHub Private Vulnerability Reporting**:

1. Open the repository's **Security** tab.
2. Click **Report a vulnerability**.
3. Provide reproduction details, impact, and affected components.

Do **not** disclose vulnerabilities in public issues, pull requests, or discussions.

## Response Process

After receiving a report, maintainers will:

1. Acknowledge receipt.
2. Assess severity and affected scope.
3. Prepare and validate a fix.
4. Coordinate responsible disclosure.

We will communicate status updates through the private advisory thread.

## Preferred Report Content

To speed up triage, include:

- A clear description of the vulnerability
- Affected versions and components
- Reproduction steps or proof of concept
- Potential impact and exploitability
- Suggested mitigation (if available)
