# Governance

[简体中文](GOVERNANCE.zh-CN.md) | English

## Model

Goyais follows an open, review-driven governance model aligned with Apache-style project practices:

- transparent discussion in issues/PRs
- consensus-first decision making
- documented technical decisions
- maintainers accountable for release quality

## Roles

### Maintainers

- Own technical direction and release decisions
- Approve high-risk and contract-affecting changes
- Enforce quality and security gates

### Reviewers / Committers

- Review design, implementation, tests, and compatibility
- Ensure docs and contracts stay consistent

### Contributors

- Submit focused, test-backed changes via PR
- Follow project standards and contribution flow

## Decision Process

1. Small changes: PR review consensus.
2. Significant changes: design discussion in issue/PR + documented decision.
3. Breaking changes: explicit maintainer approval and migration plan.

## Release Responsibility

Maintainers are responsible for:

- changelog/release notes quality
- compatibility statements
- security and rollback readiness
